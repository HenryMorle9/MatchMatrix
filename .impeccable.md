## Design Context

### Users
Developers and recruiters evaluating a portfolio project. They're scanning for technical competence, attention to detail, and polish. The interface should immediately signal "this person builds high-quality software" and invite exploration of the algorithms underneath.

### Brand Personality
**Professional, Precise, Tech-Forward.** The app should feel like a polished developer tool — clean, confident, and focused. Technical precision meets clean aesthetics. Every interaction should feel intentional and snappy.

### Emotional Goals
- "This person is technically sharp" — complexity surfaced clearly, not hidden
- "This is impressive" — the UI itself is a portfolio piece, not just a wrapper
- "I want to click around" — clean layout and clear hierarchy reward exploration

### Aesthetic Direction
- **Reference**: [hackerrank.com](https://www.hackerrank.com/) — green accent on dark backgrounds, clean sans-serif type, minimal shadows, solid panels, professional developer tool aesthetic
- **Visual tone**: Dark, clean, professional. Developer tool meets data visualization dashboard
- **Theme mode**: Dark only (intentional)
- **Color palette**:
  - Very dark navy-black backgrounds (#030914 base, #0a1628 surface, #111d2e elevated)
  - Primary brand accent: Green (#1ae05f) for CTAs, active states, section titles
  - Team 1: Light blue (#38bdf8) for data, info, Team 1 indicators
  - Team 2: Red (#ef4444) for Team 2 indicators
  - Error: Darker red (#CC0000) — visually distinct from Team 2
  - Amber/gold (#f59e0b) for warnings and "guaranteed best" algorithm
  - Cream/off-white (#ECE8E1) for body text instead of pure white
  - Subtle borders using rgba(255,255,255,0.08)
- **Typography**:
  - Outfit for all headings and body text (clean, professional)
  - JetBrains Mono for labels, data, code, nav links, buttons
  - Uppercase only on small functional labels (nav links, section titles, button text)
  - Normal case for page titles and headings
- **Motion & Animation**:
  - Fast, purposeful entrance animations (0.35s ease-out)
  - Varied animation types per context: fadeIn for text, scaleIn for panels, slideIn for CTAs
  - Not every element needs to animate — only animate what needs attention
  - SVG graph transitions for team color changes
  - No cosmetic animations (glow pulses, shimmers) without functional purpose
- **Layout patterns**:
  - Flat, solid panel backgrounds — no glassmorphism, no backdrop-filter blur
  - 4px border-radius consistently (subtle rounding, not sharp, not oversized)
  - Generous spacing — let elements breathe
  - Max-width container (7xl) for content
  - Clean borders (1px solid rgba) rather than heavy shadows

### Anti-References
- Glassmorphism / frosted glass effects — nothing blurred or translucent
- Uniform fadeInUp on every element — animations must vary and have purpose
- Oversized border-radius (16px+ pill shapes) — keep corners tight at 4px
- Red-on-red confusion — brand accent and Team 2 must be distinct colors
- Generic Bootstrap/Material UI — nothing should look like a default template
- Overly dramatic esports aesthetics — professional, not theatrical

### Design Principles
1. **Clean with confidence** — Solid backgrounds, tight borders, restrained color. Every element earns its place through clarity, not decoration.
2. **Motion as meaning** — Animations communicate state changes and guide attention, not just decorate. Varied animation types reveal hierarchy.
3. **Technical confidence** — Show complexity clearly. Monospace data, visible algorithms, graph structures front and center.
4. **Portfolio-grade polish** — Consistent spacing, aligned elements, smooth transitions. The UI itself is the portfolio piece.
5. **Color with purpose** — Green for brand/actions, blue for Team 1, red for Team 2, amber for warnings. Each color has one job.
